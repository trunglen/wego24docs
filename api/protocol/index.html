Request

Public request bao gồm các request không cần xác thực như login, reset_password, …

Private request bao gồm các request mà cần xác thực người yêu cầu. Ví dụ: update profile, đổi mật khẩu, thêm đơn hàng,… Tất cả private request cần gửi key kèm theo. Client nhận key ở trong response nếu thực hiện thành công một số api trong phần Authenticate.

Response

{
    "status": "success/error",
    "error": "if error, this is the message",
    "access_token": "if a new key was created",
    "data": "if success, this is the data"
}

Response <default có nghĩa không cần kiểm tra "data"

Response Data nghĩa là "data" trong response
